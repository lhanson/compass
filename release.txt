Release Process:

* Commit changes for release

* Update version number in project.properties, pom.xml, readme.txt,
    fork-history.txt, docs/reference/src/compass.xml(releaseInfo)
* update upgrade.txt
* Run ant clean
* Run ant release
* Sign release files and upload them to the Maven OSS staging repository:
    $version=2.2.2-ldh
    mvn gpg:sign-and-deploy-file -Durl=https://oss.sonatype.org/service/local/staging/deploy/maven2/ -DrepositoryId=sonatype-nexus-staging -DpomFile=pom.xml -Dfile=dist/compass-$version.jar
    mvn gpg:sign-and-deploy-file -Durl=https://oss.sonatype.org/service/local/staging/deploy/maven2/ -DrepositoryId=sonatype-nexus-staging -DpomFile=pom.xml -Dfile=dist/compass-$version-javadoc.jar -Dclassifier=javadoc
    mvn gpg:sign-and-deploy-file -Durl=https://oss.sonatype.org/service/local/staging/deploy/maven2/ -DrepositoryId=sonatype-nexus-staging -DpomFile=pom.xml -Dfile=dist/compass-$version-src.jar -Dclassifier=sources
* Close the staging repository on oss.sonatype.org
* Test the release using the version from the staging repository
* Promote the release to Central

* Commit version/release changes
* Add a signed tag for the release (git tag -s -m"Releasing x.x.x-ldh" x.x.x-ldh)
